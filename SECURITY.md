# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Q, please report it responsibly.

**Email:** [q.cue.2026@gmail.com](mailto:q.cue.2026@gmail.com)

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgement:** Within 48 hours
- **Initial assessment:** Within 7 days
- **Fix target:** Within 30 days of confirmation

## Disclosure Policy

- Please do **not** open public issues for security vulnerabilities
- We will coordinate disclosure with you after a fix is available
- Reporters will be credited (unless they prefer anonymity)

## Scope

This policy applies to the Q repository and all packages within it.
