⚡ Data Flows

FLOW 1 Frustration Detection
1 Observer detects rage clicks / dwell / cursor entropy on host page
2 QUI.showGreeting() opens chat, asks "Is there anything I can help?"
3 User types question → Transport.sendChat() → POST /api/v1/chat
4 Chat Agent: calls search_docs + lookup_functional_map with Claude
5 Actor.spotlight() dims page, highlights the exact element in sidebar
FLOW 2 Auto Map Discovery
1 SDK loads → DOMMapper scans all <a href> links in current DOM
2 POST /api/v1/map/discover with feature name + selector + URL
3 Backend upserts new entries (never overwrites manual ones)
4 MutationObserver + pushState patch re-scans on SPA navigation
5 Map self-heals as UI evolves — no manual maintenance needed
FLOW 3 Knowledge Ingestion
1 ingest-openclaw.sh POSTs all .md docs to POST /api/v1/ingest/text
2 Backend chunks text, calls Ollama nomic-embed-text for 768-dim vectors
3 Vectors stored in ChromaDB collection namespaced by tenantId
4 At query time: embed user question, cosine similarity search, return top-k
FLOW 4 New Customer Onboarding
1 Customer signs up → receives sdkKey + secretKey + plan tier
2 Add <script> snippet with Q.init(sdkKey) to their app
3 DOMMapper auto-builds functional map on first page load
4 Ingest product docs via API or GitHub crawler (Phase 2)
5 Q is fully operational — zero manual configuration needed
